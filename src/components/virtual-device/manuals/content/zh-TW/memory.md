用於把虛擬 ROM / RAM 接到設計上，支援檔案初始化，也支援在執行時觀察地址、輸出資料和 RAM 寫入結果。

## 模式區別

- `ROM`：只讀，提供地址輸入和資料輸出。
- `RAM`：可讀可寫，額外包含 `DIN` 和 `WE`。

切換模式後，引腳表和內部行為都會變化。

## 控制訊號約定

- `CS`：低電平有效片選
- `OE`：低電平有效讀輸出使能
- `WE`：低電平有效寫使能，僅 RAM 模式存在

如果你的設計把這些控制定義成高電平有效，必須在 RTL 裡自行取反後再連線。

## 檔案初始化支援

支援以下輸入格式：

- Vivado：`.mem`、`.coe`
- Quartus：`.mif`
- Intel HEX：`.hex`
- 原始二進位制：`.bin`
- 普通文字：預設按十六進位制解析，支援 `0x / 0b / 0o / 0d` 字首和 `@ADDR` 跳轉

這適合拿來喂字型檔、查詢表、初始化程式、測試向量和軟核 ROM 內容。

## 可觀察到什麼

- 當前總字數
- 當前被訪問的地址
- 當前輸出字
- 當前預覽視窗中的記憶體內容

對於 RAM，執行時寫入會直接反映到虛擬器件狀態裡，所以你可以邊執行邊確認寫通路是否正常。

## 適合除錯什麼

- 地址線位序是否接對
- ROM 初始化內容是否裝載正確
- RAM 寫使能時序是否正確
- `CS / OE / WE` 控制邏輯有沒有反極性或時序錯誤
