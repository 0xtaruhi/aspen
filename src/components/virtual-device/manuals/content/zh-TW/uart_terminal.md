用於和設計進行 UART 串列埠互動，既可以向設計傳送串列埠位元組，也可以解碼設計發回來的文字。

## 方向約定

- `RX` 是驅動到設計的輸入線。
- `TX` 是從設計取樣的輸出線。
- 在 `TX only / RX only / TX + RX` 三種模式下，可見引腳會跟著變化。

## 串列埠模型

- 採用典型的 **8N1** 幀格式。
- 空閒電平為高。
- 每個 bit 的持續時間由 `cycles per bit` 決定。
- 傳送和接收都依賴實時資料流，所以它本質上是“按流取樣”的 UART，而不是獨立的精確串列埠模擬器。

## `cycles per bit` 為什麼重要

- 這個值表示在資料流中，一個 UART bit 跨越多少個取樣週期。
- 它應與設計中的位週期保持一致。

## 適合做什麼

- 看 Boot 日誌、狀態列印、命令列回顯。
- 給設計傳送簡單命令、ASCII 文字或測試輸入。
- 排查串列埠傳送器、接收器、分頻器和協議 framing 邏輯。
