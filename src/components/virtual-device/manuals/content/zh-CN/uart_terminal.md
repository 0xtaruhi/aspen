用于和设计进行 UART 串口交互，既可以向设计发送串口字节，也可以解码设计发回来的文本。

## 方向约定

- `RX` 是驱动到设计的输入线。
- `TX` 是从设计采样的输出线。
- 在 `TX only / RX only / TX + RX` 三种模式下，可见引脚会跟着变化。

## 串口模型

- 采用典型的 **8N1** 帧格式。
- 空闲电平为高。
- 每个 bit 的持续时间由 `cycles per bit` 决定。
- 发送和接收都依赖实时数据流，所以它本质上是“按流采样”的 UART，而不是独立的精确串口仿真器。

## `cycles per bit` 为什么重要

- 这个值表示在数据流中，一个 UART bit 跨越多少个采样周期。
- 它应与设计中的位周期保持一致。

## 适合做什么

- 看 Boot 日志、状态打印、命令行回显。
- 给设计发送简单命令、ASCII 文本或测试输入。
- 排查串口发送器、接收器、分频器和协议 framing 逻辑。
