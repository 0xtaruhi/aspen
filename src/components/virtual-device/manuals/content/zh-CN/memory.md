用于把虚拟 ROM / RAM 接到设计上，支持文件初始化，也支持在运行时观察地址、输出数据和 RAM 写入结果。

## 模式区别

- `ROM`：只读，提供地址输入和数据输出。
- `RAM`：可读可写，额外包含 `DIN` 和 `WE`。

切换模式后，引脚表和内部行为都会变化。

## 控制信号约定

- `CS`：低电平有效片选
- `OE`：低电平有效读输出使能
- `WE`：低电平有效写使能，仅 RAM 模式存在

如果你的设计把这些控制定义成高电平有效，必须在 RTL 里自行取反后再连接。

## 文件初始化支持

支持以下输入格式：

- Vivado：`.mem`、`.coe`
- Quartus：`.mif`
- Intel HEX：`.hex`
- 原始二进制：`.bin`
- 普通文本：默认按十六进制解析，支持 `0x / 0b / 0o / 0d` 前缀和 `@ADDR` 跳转

这适合拿来喂字库、查找表、初始化程序、测试向量和软核 ROM 内容。

## 可观察到什么

- 当前总字数
- 当前被访问的地址
- 当前输出字
- 当前预览窗口中的内存内容

对于 RAM，运行时写入会直接反映到虚拟器件状态里，所以你可以边运行边确认写通路是否正常。

## 适合调试什么

- 地址线位序是否接对
- ROM 初始化内容是否装载正确
- RAM 写使能时序是否正确
- `CS / OE / WE` 控制逻辑有没有反极性或时序错误
